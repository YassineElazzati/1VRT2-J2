BLOC 01Fondamentaux de la virtualisation

Q01

Quelle est la définition correcte de la virtualisation dans le contexte des infrastructures informatiques modernes ?

La virtualisation crée des couches d'abstraction logicielles au-dessus du matériel physique pour simuler des ressources indépendantes.
Q02

Quel est l'un des principaux avantages de la virtualisation pour les entreprises qui gèrent des serveurs physiques ?

Un serveur physique héberge plusieurs VM indépendantes. La virtualisation nécessite toujours du matériel physique en dessous.
Q03

Quel pourcentage d'utilisation des ressources d'un serveur physique peut-on atteindre grâce à la virtualisation ?

Un serveur non virtualisé n'utilise que 10-20 % de sa capacité. La virtualisation permet d'atteindre 70-80 %.
Q04

Comment appelle-t-on le serveur physique qui héberge les machines virtuelles dans une architecture virtualisée ?

Le Host est le serveur physique avec le CPU, la RAM et le stockage réels, sur lequel s'exécute l'hyperviseur. Le Guest, c'est la VM elle-même.
Q05

Quel composant logiciel s'exécute sur le serveur physique et permet de créer et gérer les machines virtuelles ?

L'hyperviseur est la couche logicielle clé qui crée, gère et isole les VM en répartissant les ressources du serveur.
Q06

Quelle affirmation décrit correctement le principe d'isolation des machines virtuelles sur un même serveur physique ?

L'isolation est un avantage clé : chaque VM est indépendante, un crash n'affecte pas les autres. Le partage d'OS, c'est le principe des conteneurs (Docker), pas des VM.
Q07

Quel avantage environnemental la virtualisation apporte-t-elle aux entreprises qui gèrent de grandes infrastructures IT ?

Moins de serveurs physiques = moins d'électricité, moins de climatisation, donc empreinte carbone réduite. Attention aux mots absolus dans les distracteurs.
Q08

Quels éléments virtuels possède chaque machine virtuelle de manière indépendante dans une architecture de virtualisation ?

Chaque VM dispose de son OS, ses vCPU, sa RAM virtuelle, son disque virtuel (.vdi, .vmdk) et ses interfaces réseau virtuelles. Tout est virtuel.
Q09

Quel pourcentage de réduction des coûts d'infrastructure peut-on obtenir grâce à la virtualisation en entreprise ?

La virtualisation réduit significativement les coûts en consolidant les serveurs : 50-70 % sur le matériel, l'énergie et la maintenance.
Q10

Quel pourcentage des serveurs d'entreprise sont actuellement virtualisés selon les études du marché citées dans le cours ?

Environ 90 % des serveurs d'entreprise sont virtualisés (source Gartner).

BLOC 02Types d'hyperviseurs

Q11

Comment appelle-t-on un hyperviseur qui s'installe directement sur le matériel physique sans système d'exploitation intermédiaire ?

Le Type 1 s'installe directement sur le hardware sans OS hôte. Bare-metal = "métal nu". Type 3 et Type 0 n'existent pas.
Q12

Comment appelle-t-on un hyperviseur qui s'installe comme une application sur un système d'exploitation déjà existant ?

Le Type 2 (hosted) fonctionne comme une application classique sur Windows, macOS ou Linux. Exemple : VirtualBox.
Q13

Lequel de ces logiciels est un hyperviseur de Type 1 utilisé principalement dans les datacenters des grandes entreprises ?

ESXi est le Type 1 le plus répandu en entreprise. Les trois autres (VirtualBox, Parallels, Fusion) sont des Type 2.
Q14

Pourquoi les hyperviseurs de Type 1 offrent-ils de meilleures performances que les hyperviseurs de Type 2 en production ?

Sans couche d'OS hôte entre l'hyperviseur et le matériel, les VM accèdent plus directement au CPU, à la RAM et au stockage.
Q15

Quel hyperviseur de Type 2 est entièrement gratuit, open-source et disponible sur Windows, macOS et Linux ?

VirtualBox est l'hyperviseur Type 2 gratuit, open-source et multi-plateforme par excellence. C'est celui qu'on a installé en TP.
Q16

Quel est l'usage typique recommandé pour un hyperviseur de Type 2 dans un contexte professionnel en entreprise ?

Les Type 2 sont parfaits pour dev / test / formation sur un poste de travail. La production, c'est du Type 1.
Q17

Quel inconvénient majeur est associé aux hyperviseurs de Type 1 par rapport aux hyperviseurs de Type 2 ?

Le coût des licences (VMware), la complexité de mise en oeuvre et le besoin de matériel dédié sont les principaux inconvénients du Type 1.
Q18

Quel hyperviseur de Type 1 est une solution open-source basée sur KVM, très populaire chez les hébergeurs web ?

Proxmox VE = open-source, basé sur Debian + KVM, gratuit, très utilisé par les hébergeurs.
Q19

Quel hyperviseur est spécifiquement conçu et optimisé pour fonctionner sur les Mac avec puces Apple Silicon (M1, M2, M3) ?

Parallels Desktop est optimisé nativement pour Apple Silicon. Sur Mac M1/M2/M3, c'est lui qui offre les meilleures perfs.
Q20

Quelle est la part de marché approximative de VMware ESXi dans les grandes entreprises du classement Fortune 500 ?

VMware domine le marché entreprise avec environ 75 % de part de marché parmi les grandes entreprises Fortune 500.

BLOC 03Installation et configuration de VirtualBox

Q21

Quelle commande PowerShell permet de vérifier si la virtualisation matérielle est activée sur un ordinateur Windows ?

systeminfo | findstr /i "virtualization"
La commande LIT les informations système. Le pipe envoie la sortie à findstr qui filtre les lignes contenant "virtualization". Aucune commande depuis l'OS ne peut modifier le BIOS.
Q22

Quelle commande permet de vérifier la version de VirtualBox actuellement installée en ligne de commande ?

VBoxManage --version
VBoxManage est l'outil CLI de VirtualBox. --version affiche la version. La commande est en lecture seule.
Q23

Comment s'appelle la technologie Intel qui doit être activée dans le BIOS pour exécuter des VM en 64-bit ?

VT-x (Virtualization Technology) = support matériel Intel pour la virtualisation. L'équivalent AMD est AMD-V (ou SVM).
Q24

Que fournit le VirtualBox Extension Pack en complément de l'installation de base de VirtualBox ?

L'Extension Pack ajoute : USB 2.0/3.0, chiffrement AES, boot PXE Intel, accès distant RDP. Gratuit pour usage personnel et éducatif.
Q25

Quelle commande Linux permet de vérifier si le processeur supporte la virtualisation matérielle Intel VT-x ou AMD-V ?

egrep -c '(vmx|svm)' /proc/cpuinfo
vmx = Intel VT-x, svm = AMD-V (Secure Virtual Machine). Un résultat supérieur à 0 signifie que la virtualisation est supportée. Commande en lecture seule.
Q26

Quelle est la règle d'or concernant l'allocation de mémoire RAM aux machines virtuelles sur un serveur hôte ?

Toujours garder 40-50 % de la RAM pour le système hôte et l'hyperviseur. Dépasser cause des ralentissements (swap, freeze).
Q27

Quelle est la différence entre un disque virtuel dynamiquement alloué et un disque à taille fixe dans VirtualBox ?

Un disque dynamique de 20 GB n'occupe sur l'hôte que l'espace réellement utilisé. Un disque fixe crée immédiatement un fichier de 20 GB, légèrement plus rapide mais gourmand.
Q28

Quelle commande permet d'ajouter un utilisateur Linux au groupe vboxusers nécessaire pour les périphériques USB ?

sudo usermod -aG vboxusers $USER
Le flag -a (append) ajoute au groupe sans retirer les groupes existants. -G spécifie le groupe. $USER est le login courant. Sans le -a, on perdrait tous les autres groupes.
Q29

Quel format de fichier de disque dur virtuel est le format natif et spécifique à Oracle VirtualBox ?

VDI = format natif VirtualBox. VMDK = VMware. VHD/VHDX = Hyper-V. ISO = image CD/DVD (pas un disque dur virtuel).
Q30

Combien de cœurs CPU maximum devrait-on allouer aux VM sur un ordinateur personnel ayant 4 cœurs physiques ?

Règle : max 50-75 % des cœurs aux VM. Sur 4 cœurs, cela donne 2 à 3 cœurs pour les VM, en gardant une marge pour l'hôte.
